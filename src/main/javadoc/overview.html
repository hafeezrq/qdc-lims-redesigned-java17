QDC LIMS System Overview

QDC LIMS is a desktop-first Laboratory Information Management System built on Spring Boot, Spring Data JPA, and JavaFX. It manages patient registration, lab orders, result entry, reporting, inventory consumption, and financial operations in a single application.

System Map

  • Root: com.qdc.lims
  • Configuration: com.qdc.lims.config
  • Domain Entities: com.qdc.lims.entity
  • Repositories: com.qdc.lims.repository
  • Services: com.qdc.lims.service
  • UI Root: com.qdc.lims.ui
  • UI Controllers: com.qdc.lims.ui.controller
  • Navigation: com.qdc.lims.ui.navigation
  • Sessions: com.qdc.lims.ui.session
  • UI Utilities: com.qdc.lims.ui.util
  • General Utilities: com.qdc.lims.util

Architecture At A Glance

  • UI layer: JavaFX controllers under com.qdc.lims.ui.
  • Service layer: Business logic under com.qdc.lims.service.
  • Persistence layer: JPA entities under com.qdc.lims.entity and repositories under com.qdc.lims.repository.
  • Configuration/utilities: Supporting code under com.qdc.lims.config and com.qdc.lims.util.

Core Functional Areas

  • Patients and Orders: Patient registration, lab orders, and result workflows.
  • Test Catalog: Test definitions, departments (categories), panels, and reference ranges.
  • Reporting: PDF report generation and export helpers.
  • Inventory: Inventory items, consumption recipes, purchases, and supplier ledgers.
  • Finance: Payments, commissions, and financial summaries.
  • Security and Sessioning: User authentication, password policy, and session/window management.

Suggested Reading Order

  1. Application entry points: com.qdc.lims.DesktopApplication and com.qdc.lims.QdcLimsApplication.
  2. Domain model: packages under com.qdc.lims.entity.
  3. Business logic: packages under com.qdc.lims.service.
  4. UI flows: controllers under com.qdc.lims.ui.controller.
